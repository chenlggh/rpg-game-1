# RPG 游戏实施计划

## 概述

本计划基于 [游戏设计文档](../specs/2026-06-08-rpg-game-design.md)，按照 TDD（测试驱动开发）原则，分阶段实现一款短篇 JRPG。

## 阶段一：地图行走 + NPC 对话系统

### 1.1 核心场景搭建

| 序号 | 任务 | 验证方式 |
|------|------|----------|
| 1.1.1 | 创建 `project.godot` 基础项目配置 | Godot 4 打开不报错 |
| 1.1.2 | 创建 TileMap 场景 `scenes/world/overworld.tscn` | 编辑器可预览瓦片地图 |
| 1.1.3 | 创建玩家角色节点 `Player.gd`（继承 CharacterBody2D） | 移动方向输入响应正常 |
| 1.1.4 | 实现 4 方向移动 + 碰撞检测 | 碰到墙壁不穿过 |

### 1.2 对话系统

| 序号 | 任务 | 验证方式 |
|------|------|----------|
| 1.2.1 | 创建对话节点 `DialogueBox.gd`（Control 子节点 + Label） | 对话框弹出显示文本 |
| 1.2.2 | 实现 NPC 交互区域 `Area2D`（靠近时按 E 键触发） | 按键后对话框弹出 |
| 1.2.3 | 创建剧情数据结构 `scripts/data/dialogues.gd` | 可加载多段对话文本 |
| 1.2.4 | 实现选项分支（二选一影响剧情走向） | 选项点击后进入不同分支 |

**TDD 测试要点：**
- `test_player_movement.gd` — 模拟 4 方向输入，验证坐标变化
- `test_collision.gd` — 创建碰撞区域，验证玩家无法穿过
- `test_dialogue_flow.gd` — 验证对话文本按序显示、选项分支跳转正确

### 预估工作量：2-3 天

## 开发规范

1. **TDD 原则**：每个功能模块先写测试用例，再实现逻辑。
2. **YAGNI 原则**：不做多余的系统（无公会、无多人、无小游戏）。
3. **占位先行**：原型阶段用矩形/圆形占位美术，后期替换为像素精灵。
4. **单存档位**：初期仅支持一个存档，简化数据管理。
5. **中文文档**：所有设计和注释使用中文。
